# Repository Instructions

- Use `main` for repository changes unless the user explicitly asks for a separate branch or pull request.
- Do not create PR-only branches for routine work in this repository.
- Keep rollback points as ordinary commits on `main`.
- Do not create generated work artifacts inside this repository. Keep audits, reports, screenshots, verification notes, temporary files, and cleanup outputs outside the public repo, preferably under `C:\Dev\docs` or `C:\Dev\reports`.
- Treat every tracked file in this repository as public. Do not add secrets, private notes, local-only reports, dependency dumps, or Codex scratch files to the repo tree.
