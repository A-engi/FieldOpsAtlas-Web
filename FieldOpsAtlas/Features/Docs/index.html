Docs

Field reference library

Documents are split by job phase and evidence type. Access control, site condition reports, rack walkthroughs and admin are kept separate so the app does not become one long mixed checklist.

Job workflow

Before leaving Fault call time, SLA, outage windows, customer method, kit, PPE, spares, rota and calendars. Site access Keys, gates, parking, CCTV, access contact and navigation belong with the site map record. Site reporting Overgrowth, blocked access, vehicle scratches, safety observations and photos. After the job Mileage, timesheets, timecards, spares return, inventory, stock and deliveries.

Technical reference

Service layout Building, room, rack, power, network, RF paths, incoming services and outgoing dependencies. Manuals and methods Customer-specific work methods, PM history, common faults, manuals and escalation notes. RF services DTT, DAB, FM, path, parent/sister site and service parameter records. Field tools dB, VSWR, sunrise/sunset, circuit basics, location capture and route signal logging.

Data rule

Pull job facts from the source website The app should read the live job record, change request, calendars and stock systems instead of storing private job information in this public repo.
Use local drafts only for missing field data Engineer notes, diagrams, marked photos, scans and aliases should be local/private until they are sent back to the correct system.
